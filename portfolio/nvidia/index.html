Asynchronous multi-GPU training for deep learning

Asynchronous optimizer for a distributed data parallel training regime

@ Nvidia | Summer, 2020.

Team Members: Pranav Mahajan (intern), Supratik Majumder (mentor), Santosh Madugula (mentor), Raghuram L (manager).

I implemented an asynchronous stochastic gradient descent (SGD) for a distributed data parallel training regime for the single-shot detector (SSD) network and quantified its performance. Please find my final presentation and a detailed report for more details. I’ve also been fortunate to receive a pre-placement offer by the end of this internship.

Overview

Distributed data-parallel training has been using synchronous training for the past years, we can see in the recent MLPerf submissions as well. In this work, we try to understand the performance scalability issues with synchronous training and how researchers have attempted to solve it via asynchronous techniques. Most of the prior methods for asynchronous methods use the concept of parameter servers, which has its drawbacks, such as communication bottleneck with the server. We devise and implement a strategy for basic asynchronous training (without the use of parameter servers) for our current setup, where gradient sharing takes place using all reduce. We compare our implementation of asynchronous SGD with synchronous SGD in terms of total step time and convergence. We quantify the performance in terms of step time required for a forward pass, backward pass + all reduce, optimization step, and quantify jitter for synchronous training on the Single Stage Detector (SSD) network. We also compare the decrease in loss function value per epoch between synchronous SGD and asynchronous SGD. We see the asynchronous SGD loss starts deviating from the synchronous SGD, from the second epoch onward for the SSD network. We conclude by discussing the possibilities of future work.
